# DimOS 项目概览

本文用较少篇幅说明 **DimOS（Dimensional Operating System）** 是做什么的、由哪些部分组成，以及从哪里继续阅读。适合第一次接触本仓库的读者。

## 它是什么

DimOS 是一套面向**通用机器人**的运行与开发框架：把感知、规划、控制、与大模型智能体等子系统，用**类型化的数据流**组合在一起，在真机、仿真或录播数据上运行。模块在独立进程中执行，通过 LCM、共享内存、ROS 等传输层交换消息；用 **Blueprint（蓝图）** 描述「装哪些模块、如何连接」；需要自然语言操控时，用 **Skill（技能）** 把可执行动作暴露给 LLM，并可配合 MCP 服务对外提供工具接口。

项目默认以 Python 为主要开发语言，并提供 `dimos` 命令行来运行预置蓝图、查看日志、与智能体对话等。

## 核心概念（五个词）

| 概念 | 含义（一句话） |
| --- | --- |
| **Module（模块）** | 机器人上的子系统，用 `In[T]` / `Out[T]` 声明输入输出数据流。 |
| **Blueprint（蓝图）** | 若干模块的装配方案；通常用 `autoconnect(...)` 按名称与类型自动接线。 |
| **Skill（技能）** | 标有 `@skill` 的方法，可被智能体当作工具调用（需完整文档字符串与类型注解）。 |
| **Transport（传输）** | 模块之间的通信实现，如默认的 LCM、大图用 SHM、与 ROS 互通等。 |
| **GlobalConfig** | 全局配置（如 `robot_ip`、`simulation`、`replay`、可视化方式等），可被 CLI 与环境变量覆盖。 |

更细的比喻与流程说明见 [教程第 0 章 · 整体架构](tutorial/00-overview.md)。

## 硬件与能力（粗线条）

仓库内已集成或演示过多类平台，包括但不限于：四足（如 Unitree Go2）、人形（如 Unitree G1）、机械臂（如 xArm）、无人机（MAVLink / DJI）等；具体成熟度以各子目录与 capability 文档为准。导航建图、感知、空间记忆、遥操作、Web 可视化等能力在 `docs/capabilities/` 与根目录 [README.md](../README.md) 中有入口。

## 怎么上手

1. **读仓库约定与命令**：根目录 [AGENTS.md](../AGENTS.md)（面向贡献者与高级用法，含 Quick Start 与模块/技能约定）。  
2. **安装与环境**：见 [README.md](../README.md) 的 Installation，以及 [安装文档](installation/ubuntu.md) / [系统要求](requirements.md)。  
3. **无硬件试跑**（示例）：在已配置好的环境中可尝试 `dimos --replay run unitree-go2` 等命令（详见 AGENTS.md 与 README）。  
4. **深入概念**：[usage/README.md](usage/README.md)（Modules、Streams、Blueprints 等索引）。
